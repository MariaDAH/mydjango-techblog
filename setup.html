Setting up an AWS environment
Create manually an environment in AWS Cloud9
Open terminal and type:
wget -q https://git.io/v77xs -0 /tmp/setup-workspace.sh && source /tmp/setup-workspace.sh
Create virtual enviorment installs automatically all that is needed to set up django and activate the virtual environment.
mkvirtualenv foo
Install django
pip3 install django
If necessary downgrade to a earlier version
pip install django==1.11
Create the first requirements
pip freeze > requirements.txt
Create a django project in the root directory and make the manage.py and executable file changing the permissions and execute migrate to initialize the database
django-admin startproject blog .
chmod +x manage.py
./manage.py migrate or python3 manage.py migrate To access sqlite command line you run sqlite3 db.sqlite3
select * from django_migrations
Run the application on the server
python3 ./manage.py runserver $IP:$PORT
It will get a DisallowedHost error add the django server hostname to AllOWED_HOST variable in settings.py
Initialize git repository
git init
Create a gitignore file
echo -e "*.sqlite3\n*.pyc\n.~c9\n__pycache__/" > .gitignore or
echo -e "*.sqlite3\n*.pyc\n.c9\n__pycache__/" > .gitignore
Create a new repository in GitHub web UI
Come back to terminal and push the changes from local to remote using SSH protocol
git remote add origin git@github.com:MariaDAH/mydjango-techblog.git
git push -u origin master
Make sure you previously generated a SSH key in your GitHub account
Synchronize with Travis for continuous integration
